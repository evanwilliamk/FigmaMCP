Accessibility - Color Contrast (Bluelines)

WCAG AA Standard: Normal text requires 4.5:1 contrast ratio. Large text (18px+ or 14px+ bold) requires 3:1 contrast ratio.

Contrast Analysis

Foreground Background Ratio Normal Text Large Text Example
#2C2C2C #FFFFFF 12.6:1 AAA AAA Primary text
#6B6B6B #FFFFFF 5.7:1 AA AAA Secondary text
#1473E6 #FFFFFF 4.5:1 AA AAA Link text
#FFFFFF #2C2C2C 12.6:1 AAA AAA Button text
#FFFFFF #1473E6 4.5:1 AA AAA Progress bar
#FFFFFF #2D9D78 3.9:1 FAIL AAA Success (large text only)
#FFFFFF #E34850 4.5:1 AA AAA Error state
#BCBCBC #F5F5F5 1.9:1 FAIL FAIL Disabled (intentional)
Note on Disabled States: Disabled elements are exempt from contrast requirements per WCAG guidelines, as they are not intended to be interacted with.
Success State Note: The green success color (#2D9D78) should only be used with bold text at 14px+ or regular text at 18px+ to meet AA standards.

ARIA Best Practices

Pattern Implementation Purpose
Native HTML First Use <button>, <input>, <label> Built-in accessibility, less code, better support
Decorative Icons aria-hidden="true" Hide non-essential visuals from screen readers
Functional Icons aria-label="Close" Provide text alternative for icon-only buttons
Live Regions aria-live="polite" Announce dynamic content changes
Landmarks role="region" with aria-label Help users navigate page sections
