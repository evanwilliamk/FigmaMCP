6. Full Panel — All States

StateWhat ChangesAccessibility Notes
Default
No platform, no interaction		 Button disabled. Note shows "single file". File size shows default estimate. Count badge hidden. Recommended badge shows (MP4 + 1080p default). All controls focusable. Reading order: Title → description → info bar → platform group → resolution → format → note → button.
Platform Selected (1) Tile selected state. Count badge shows "(1 selected)". Button enabled → "Download (1 platform)". Recommended badge contextualizes. Count badge live region announces. Button label change read on focus.
Platform Selected (Multi) Multiple tiles selected. Count badge updates. Note switches to zip packaging. File size shows multiplied estimate. Button: "Download (N platforms)". Note content changes — not announced (role="note" not live). Acceptable as user can re-read. File size change in aria-live region announces estimate.
Format/Resolution Changed File size estimate updates. Recommended badge shows/hides. Picker displays new value. File size live region announces new value. Recommended badge live region announces if it appears.
Exporting Button → loading. Progress bar appears. Platform tiles and pickers disabled (pointer-events none + opacity). Progress label updates per platform. Progress bar aria-valuenow updates. Progress label changes announced by progress region. Focus stays on button area. Disabled controls not keyboard-reachable.
Export Success Progress bar disappears. Success banner appears. Button → green "Export complete". Download link active in banner. Banner announced immediately via aria-live="polite". Focus not forcibly moved — user completes reading. Download link is new keyboard target.
Export Error Progress bar disappears. Error banner appears. Button → red "Export failed — Try again". Error banner aria-live="assertive" interrupts. Focus not moved but next Tab will reach retry link. Both button and banner retry link work.
Reset / Retry All states reset to idle with current selections preserved. Banners hidden. Button re-enabled. Banners hidden via display:none — removed from AT. No announcement needed on hide.

11. Edge Cases & Open Questions

ScenarioCurrent BehaviorRecommendation
All platforms selectedAll 7 tiles selected, zip note shown, large file estimate.Consider a "Select all / Clear all" control for efficiency. Not in current design.
Platform + format incompatibilityNot yet implemented. All formats always shown.Disable incompatible formats with tooltip. Announce via aria-describedby.
Export in progress — user tries to close panelNot handled in prototype.Show confirmation dialog: "Export in progress. Are you sure you want to cancel?" role="alertdialog".
Very long platform nameText truncates in tile (overflow:hidden).Tile aria-label always carries full name. Tooltip on hover for visual users.
File size exceeds platform limitNot yet implemented.Show inline warning adjacent to file size value. Announce via aria-live="polite".
Network failure mid-exportError state shown.Error message should be specific where possible ("Connection lost") vs generic. Retry resends.
Screen width < 320pxPanel fixed at 344px — may clip.Ensure minimum 320px viewport support per WCAG 1.4.10 (Reflow).
High contrast modeNot specifically tested.Use CSS forced-colors: active media query. Replace background-color selection indicators with border-based ones.
